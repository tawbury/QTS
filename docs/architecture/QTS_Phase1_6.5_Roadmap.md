확인했습니다. **1번이 1차 목표**로 확정되었습니다.
즉, **QTS 코어 구현을 완결하고 ops는 “연결 가능한 상태”까지만 도달**시키는 것이 목표입니다.

아래는 이 전제를 기준으로 **로드맵을 재정렬·압축한 실행 중심 요약**입니다.

---

## 🎯 1차 목표 재정의 (고정)

> **QTS(Core)가 독립적으로 자동 실행 가능하고,
> ops는 필요 시 연결만 하면 되는 상태**

* QTS: 주체(Subject)
* ops: 외부 파트너(Optional Integration)
* 수익 최적화/AI 고도화: **범위 밖**

---

## 📌 1차 목표 기준 로드맵 (압축판)

### Phase 3 — Execution 연결 완성

**결과물:** “한 번의 주문을 끝까지 수행 가능”

* Order Adapter
* Execution Route
* 실주문 제외 E2E

➡ **QTS 단독 실행 가능성 확보**

---

### Phase 4 — Controlled Automation

**결과물:** “조건부 자동 실행”

* 실행 루프
* 하드 가드(횟수/시간/손실)
* 수동/자동 모드 분리

➡ **QTS 자동 실행 개시**

---

### Phase 5 — Strategy Binding

**결과물:** “관측 → 판단 → 실행”

* Signal → Intent 바인딩
* 전략 연결(비최적화)
* 전략 단위 테스트

➡ **QTS 자동 매매 구조 완성**

---

### Phase 6 — Risk & Safety (최소 필수)

**결과물:** “망가지지 않음”

* 손실/빈도/중단 규칙
* Kill-switch
* Fail-safe 테스트

➡ **사람 개입 없는 안전성 확보**

---

### Phase 6.5 — ops 연결 준비 (최소)

**결과물:** “연결해도 안전”

* Backup-first retention
* 자동 클린업
* 운영 로그 고정

➡ **ops는 ‘연결 가능’ 상태**

> ⚠️ ops는 **주체가 아님**
> 연결·분리는 언제든 가능해야 함

---

## ⛔ 1차 목표에서 의도적으로 하지 않는 것

* 수익률 최적화
* 멀티 브로커 확장
* 전략 학습/AI
* ops 자동화 주도

---

## 🧭 판단 기준 (앞으로의 의사결정 룰)

* **QTS가 스스로 돌아가는가?** → YES면 진행
* **ops 없으면 안 되는가?** → NO여야 함
* **구조 변경이 필요한가?** → Phase 종료 후만 허용
