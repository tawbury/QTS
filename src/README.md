# src/ — 소스 코드

QTS 프로젝트의 핵심 소스 코드가 위치한 디렉토리입니다.

## 모듈 구조

| 모듈 | 설명 |
|------|------|
| **automation** | 자동화된 작업 및 스케줄러 관리 |
| **backup** | 데이터 백업 및 복구 메커니즘 |
| **db** | 데이터베이스 연결, 리포지토리 패턴 구현 (Google Sheets 등) |
| **decision_pipeline** | 전략적 의사결정 흐름 제어 |
| **maintenance** | 시스템 유지보수 및 정리 작업 |
| **monitoring** | 시스템 상태 모니터링, 로깅, 알림 |
| **observer_client** | 외부 Observer로부터 시장 데이터를 수신하는 클라이언트 |
| **pipeline** | ETEDA 파이프라인 실행 로직 |
| **provider** | 외부 API (KIS, Kiwoom) 연동 및 어댑터 |
| **qts** | QTS 애플리케이션 핵심 설정 및 로더 |
| **retention** | 오래된 데이터의 보존 및 삭제 정책 |
| **risk** | 리스크 관리, 한도 체크, 게이트키퍼 |
| **runtime** | 애플리케이션 실행 진입점 및 런타임 환경 구성 |
| **safety** | 시스템 안전장치 (Fail-Safe, Circuit Breaker) |
| **shared** | 프로젝트 전반에서 사용되는 공통 유틸리티 및 상수 |
| **strategy** | 매매 전략 엔진 및 시그널 생성 로직 |

## 개발 참고

새로운 기능을 추가할 때는 해당 기능의 성격에 맞는 모듈에 코드를 작성하고, 필요한 경우 `tests/` 디렉토리에 테스트 코드를 작성하십시오.
